# ofxGgmlRag v1.1.0 Draft

## Compatibility

- openFrameworks: `of_v20260406_vs_64_release`
- minimum `ofxGgmlCore`: `1.0.1`
- `OFXGGML_RAG_VERSION_STRING`: unchanged until release tagging

## Highlights

- Added deterministic local text-corpus loading for user-provided `.md` and
  `.txt` files.
- Added corpus options, stats, warnings, and result structs.
- Added `ofxGgmlRagUtils::loadTextCorpus(...)` for converting local files into
  `ofxGgmlRagDocument` values.
- Added `ofxGgmlRagUtils::retrieveTextCorpus(...)` for one-call source-root
  retrieval.
- Added offline HTML-to-document helpers for already-fetched web pages while
  keeping live crawling/fetching out of the deterministic helper layer.
- Added offline HTML link-frontier extraction with same-origin filtering and
  relative URL resolution for future crawler wiring.
- Added offline HTML link-frontier planning diagnostics with per-link
  accept/skip reasons.
- Added allowed/excluded URL prefix filters for scoped offline HTML link
  frontiers.
- Added robots-meta-aware offline HTML ingestion policy for `noindex` and
  `nofollow` directives.
- Added deterministic offline robots.txt parsing and longest-prefix URL policy
  checks for already-fetched robots text.
- Added robots.txt-aware offline HTML link-frontier filtering for callers that
  provide already-fetched robots text.
- Added offline HTML page-batch ingestion snapshots that produce documents,
  aggregate link frontiers, stats, and warnings from already-fetched pages.
- Added the stateful `ofxGgmlRag` facade for app-facing corpus loading,
  in-memory document search, retrieval summaries, text reports, and JSON
  reports.
- Added citation-grounded prompt handoff structs and helpers for the future
  model-backed answer layer.
- Added extractive citation-backed answer drafts for UI previews that need a
  concrete answer-shaped object before model-backed generation exists.
- Added a pluggable prompt-generator callback on the `ofxGgmlRag` facade for
  model-backed cited answers from app-provided ggml/GGUF generation backends.
- Added one-call facade helpers for retrieval plus configured model-backed
  answer generation.
- Added stopword-aware query refinement, query variants, source quality hints,
  and cosine/vector search helpers as dependency-light RAG pipeline parity work.
- Added local exact-quote citation search with intent detection, markdown
  cleanup, confidence scoring, source credibility, and source diversity metrics.
- Added facade-level retrieval caching with cache-hit diagnostics and
  document-change invalidation.
- Added facade-level retrieval cache introspection for app UI state.
- Wired corpus readiness into `doctor-rag.ps1`.
- Wired corpus retrieval into `run-rag-runtime-smoke.ps1` when `-SourceRoot` is
  provided.
- Updated the ImGui-backed `ofxGgmlRagSearchExample` to run deterministic
  retrieval against a user source root, with an in-memory fallback when no
  source root is set.
- Added fresh clone and corpus smoke instructions in `docs/GETTING_STARTED.md`.

## Runtime Scope

This draft release scope includes deterministic text-file ingestion into the
existing in-memory request-boundary retrieval helpers. It can load supported
local text files, convert already-fetched HTML pages into documents, extract a
deterministic HTML link frontier with optional robots.txt policy filtering and
per-link planning diagnostics, build offline page-batch snapshots, chunk
documents, search them, assemble citations/context, and emit text or JSON
retrieval reports, prompt handoff text, extractive answer drafts, app-provided
model-backed answer callbacks, and local exact-quote citation search results.

It does not include live web crawling/fetching, embedding generation, vector
indexes, reranking models, built-in llama.cpp generation, project-memory
persistence, model downloads, or generated corpus/index artifacts.
