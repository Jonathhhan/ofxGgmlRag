# ofxGgmlRag v1.1.0 Draft

## Compatibility

- openFrameworks: `of_v20260406_vs_64_release`
- minimum `ofxGgmlCore`: `1.0.1`
- `OFXGGML_RAG_VERSION_STRING`: unchanged until release tagging

## Highlights

- Added deterministic local text-corpus loading for user-provided `.md` and
  `.txt` files.
- Added corpus options, stats, warnings, and result structs.
- Added `ofxGgmlRagUtils::loadTextCorpus(...)` for converting local files into
  `ofxGgmlRagDocument` values.
- Added `ofxGgmlRagUtils::retrieveTextCorpus(...)` for one-call source-root
  retrieval.
- Added the stateful `ofxGgmlRag` facade for app-facing corpus loading,
  in-memory document search, retrieval summaries, text reports, and JSON
  reports.
- Added citation-grounded prompt handoff structs and helpers for the future
  model-backed answer layer.
- Added extractive citation-backed answer drafts for UI previews that need a
  concrete answer-shaped object before model-backed generation exists.
- Added stopword-aware query refinement, query variants, source quality hints,
  and cosine/vector search helpers as dependency-light RAG pipeline parity work.
- Wired corpus readiness into `doctor-rag.ps1`.
- Wired corpus retrieval into `run-rag-runtime-smoke.ps1` when `-SourceRoot` is
  provided.
- Updated the ImGui-backed `ofxGgmlRagSearchExample` to run deterministic
  retrieval against a user source root, with an in-memory fallback when no
  source root is set.
- Added fresh clone and corpus smoke instructions in `docs/GETTING_STARTED.md`.

## Runtime Scope

This draft release scope includes deterministic text-file ingestion into the
existing in-memory request-boundary retrieval helpers. It can load supported
local text files, chunk them, search them, assemble citations/context, and emit
text or JSON retrieval reports, prompt handoff text, and extractive answer
drafts.

It does not include web crawling, embedding generation, vector indexes,
reranking models, llama.cpp generation, project-memory persistence, model
downloads, or generated corpus/index artifacts.
